User Manual

Here you can read about how the user is supposed to use our web app

Figure 1: The Welcome Screen of Ammoenio


A new user can click on “SignUp” to create a new account. A recurring user can click on “Login” and log-in with their credentials to access the website. In the latter case the platform automatically recognizes whether the user has a business or investor account. If a user wishes to create an account they can choose between creating one of the aforementioned accounts. A business account only allows the creation of companies and investing in them, not displaying all the other companies in the platform. An investor account allows a user to view all the available companies, invest in them, and buy or sell shares.


Figure 2:The Sign-Up Screen


All the companies that exist on the Ammoenio Platform can be seen on the dashboard. From there a user can select a company they wish to invest in, and enter an amount of money they wish to invest in the company. They will then be taken to a screen showing a detailed breakdown of their costs, including how many shares they can buy and what the fees are (included in the price the user provided). The user can then proceed to check-out and choose one of the 5 available payment methods (which are dummy methods for now, as the user doesn’t actually pay real money in this proof of concept).


Figure 3:The Buying Shares Screen


Figure 4:The Checkout Screen


When the user wants to buy shares, they can also see other users selling shares for that company. They can then decide if they want to buy directly from the company or from another user.


A user can choose to sell a number of shares for a company. This can be performed by going to “My Investments”, selecting the particular company and clicking on “Sell Shares”.

Figure 5: My Investments for a particular Company


Figure 6: My Investments Screen


For a business user, the “Explore” tab is replaced by “Create”, allowing the creation of companies. A business owner can create a company by going through the form provided by Ammoenio and specifying details about the company, including name, share information and contact details.


Figure 7: Creating a Company


It is compulsory to enter all fields, except for links to social media, and we require the upload of both a cover image and a logo. To do this, the user must browse his files, select the desired one, and press on upload. Once the upload successfully takes place the button will turn green.


Finally, when they click on “Done” the shares will be up for sale.


For business users, the “My Investments” tab is also replaced by “My Companies” tab. Business users will be able to see a preview of their company as seen by the public and, in later versions, will be able to modify this data. The “Buy Shares” button is not active as this is a preview page.


There is also an “Info” page including details about their shares and transactions. This includes the amount of shares owned, the amount sold and the percentage sold. We wish to give business the clearest possible overview of their company’s investment data.


Figure 8: Share Information


The business owner can issue more shares if they wish to, directly from here. To do this the user must click on “My Companies” and then on the “Info” button of the company they want. To issue more shares they simply need to click on the “Issue more shares” button and enter the amount of shares they want to issue to the market.




System Manual

Here you can read about how to deploy the project on to a server.

Not much is needed to be able to run the app. All libraries are included in the projects. Certain tools are however necessary. To be able to run the app, you need to have the latest versions of NodeJS and NPM installed as well as Nodemon (can be installed via NPM). Git is not necessary for debugging and testing, though it is what we used to always pull the latest version from the repository. Tmux is not necessary, but we do recommend installing it as it is needed for running one of the scripts if needed. To install all these tools, we recommend running the InitialiseMachine.sh script (type “./InitialiseMachine.sh” in the terminal; make sure the permissions for executing are set appropriately) located in AmmoenioApp/Scripts/. This script will install all the aforementioned tools automatically. We have written all the bash scripts in order to make settings up the server and the debugging and testing of the application as easily as possible.


To run the app, all that is needed is to run: RunAmmoenioOnPC.sh located in the same folder as the previous script. This will start the TestRPC client and then it will start the NodeJS server application after a delay of 8 seconds (needed to load the RPC client). Alternatively, RunAmmoenioOnServer.sh can be runned. This scripts is specifically designed to be run on servers that do not provide as user interface (or a graphical server in Ubuntu’s case). It can be used on the local computer nonetheless provided Tmux is installed. Alternatively, the two can be runned manually: TestRPC can be runned from StartTestRPC.sh or from AmmoenioApp/Ethereum Component/TestRPC/bin/testrpc (type“./testrpc”);the NodeJS server can be ran from AmmoenioApp/Back-End/Server.js with “sudo nodemon Server.js” (“sudo” is needed as the Server.js listens to port 80 which is reserved).


Note: Certain particular system specifications might render some scripts unable to run. For example, nodemon was not running in sudo mode on a computer we tested. Some scripts will also not work on Mac due to gnome-terminal not being installed. That’s why we recommend Ubuntu 16.10 and the latest versions of NodeJS and NPM.


If the computer used also encountered an issue with NodeJS and NPM (usually due to not being installed right or not having the last version), a “listen EADDRINUSE ::: 80”error will be thrown.


You can choose to run the server on a non-reserved port, e.g. port 8000. To do this, you need to edit the file Server.js and change the line:

const PORT_TO_RUN = '80';

to something like

const PORT_TO_RUN = '8000';

(keep in mind port 8545 needs to be used by TestRPC).



TestRPC is using other 3rd party libraries that are specifically compiled for Ubuntu 16.04 or later, so they will not work on Macs. A second RPC client called EtherSim is provided that can be ran on Macs as well, but this does not seem to be so reliable as TestRPC and is largely outdated.


Also make note that when restarting the RPC client, all data stored in the local blockchain is deleted and the accounts are reset. That means that all the previous user accounts and companies are unusable. This will not be a real problem once the project is released as that one will use the real Ethereum Blockchain.


Server.js also has two modes of running. Running it with “nodemon Server.js” will launch make the server application run on normal mode: it will deploy a new controller and it will inform that it should start with a fresh user list. Alternatively, it can be runned on reload mode via “nodemon Server.js reload” (this is the default way in which the server application runs if making use of the provided scripts. The reload mode/function is useful if the server application has crashed and we do not want to loose all the previous data; this will look for the backup file stored in Backups and it will reload the user list and the controller contract. Please make note that this only works if the RPC client was not restarted. If the client is restarted, all the previous data is lost. Created user accounts and companies will still remain on the server though. When running RunAmmoenioOnPC.sh or RunAmmoenioOnServer.sh, they will delete the backup file and then start the server application on reload mode so that if the server application crashes, it can be easily restarted and reload the contracts and user list. Nodemon is specifically useful as it can easily restart the server application.


Please also note that only one RPC client can be running and only one instance of it. This is because the RPC client is a server application as well listening on port 8545 so it will reserve the port for itself and it will not allow other applications to run on it.


After everything is up and running in the terminals, visit localhost:PORT_NUMBER (eg. localhost:80) to access the website.


An alternative way to test the platform is to visit our website www.ammoenio.trade which is already set-up with 3 companies.
