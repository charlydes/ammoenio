APIs

Web3JS API (Web3 JavaScript Ðapp API) [1]

Web3JS enables communication between the NodeJS server and the Ethereum Blockchain/RPC client.


CoinMarketCap API [2]

This API is used for getting the conversion rate for ETH-GBP in real time.




Tools

TestRPC [3]

TestRPC is a tool that creates a simulated version of the Ethereum Blockchain. TestRPC is a public tool and library, but we are making use of a customised version of this RPC client. We have increased the gas limit from 4 million gas to around 10 million gas and the number of available accounts (wallets) from 10 to 201 (one controller account and 200 that can be used by the users). We have also changed the default ETH balance from 100 ETH (approximately 350 GBP) to approximately 10 000 000 000 ETH (approximately 35 billion GBP) to allow for comprehensive testing and debugging.

In the actual project, TestRPC would be replaced by the real Ethereum Blockchain and the use of ETH would be replaced by GBP. ETH would only be used for processing and completing transactions which will take from 5 up to 20 minutes, meaning changes in conversion rate should have almost no effect on the user (we expect no steep changes in a 5-20 minutes time period). Both the user and the company would have GBP balances.


Nodemon [4]

We used nodemon for running the server application. This enables easy restarting of the server in case of crashing due to an error.


Digital Ocean Droplets [5]

Digital Ocean’s virtual machine (droplet) hosting services were used to host our application.


Cloudflare [6]

The DNS routing, browser caching (4 hours caches), server caching, encryption, the SSL certificate, HTTPS enforcing, firewall, traffic management, access rules, static pages and DDoS protection is provided by Cloudflare.




Frameworks

AngularJS [7]

Angular is a JavaScript framework that allows for $scope variables and, therefore, a direct interaction with the DOM. We chose this framework because of its fast and simple connection with the Firebase database through the AngularFire library.




Libraries

Zeppelin Solidity / OpenZeppelin [8]

We based our smart contracts on the Zeppelin Solidity library to ensure that they were safe. Zeppelin Solidity is a library for writing secure Smart Contracts on Ethereum. The library allows you to build distributed applications, protocols and organizations using common contract security patterns (See Onward with Ethereum Smart Contract Security) in the Solidity language.


Bootstrap [9]

We used bootstrap on the front end to facilitate some design processes. This library provides css classes for different elements (eg. buttons, forms etc.).


Angular Fire [10]

Angular Fire is a powerful Angular JS library that make querying the Firebase database easier. With simple methods such as $firebaseObject() and $firebaseArray(), we could easily store database information in $scope variables, ready to use in the AngularJS logic and in the HTML front-end.


ExpressJS [11]

ExpressJS was used for routing and HTTP requests handling. ExpressJS is essential for communicating with the AngularJS web app.


Solidity Compiler (SolC) [12]

SolC was used for compiling the contract after parsing them


Contracts Creator (eth-new-contract) [13]

Eth-new-contract is used for deploying contracts to the Blockchain/RPC client via the Web3JS API.


Locks [14]

Locks are Asynchronous locks used for locking reading and writing of files.


Body-parser [15]

Body-parser is a parsing middleware used for JSON parsing of the date in the HTTP requests.
