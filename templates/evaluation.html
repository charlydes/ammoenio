Critical Evaluation of the Project

Here you can read our critical evaluation of the project.

Domain Critical Evaluation
Architecture Design We are very happy with our robust design that has proved efficient, but also easy to migrate to alrger systems. Having all the data stored externally (external Firebase database and on the Blockchain) provided very easy scalability which is essential if we would want to transform this project into a business.
User Interface Design and User Experience We have achieved a very good user experience and a very robust, simple, but also good looking interface design. We have led several surveys with out colleguaues, but also with our client, which helped us get to where we are.
Functionality Functionality is very good as we have achieved all the Must Have, Should Have and Could Have requirements from the MoSCoW table. We would have preffered though to also do most of the Would Have features as well. Nonetheless, we are very happy with our progress.
Stability After extensive testing (and bug fixing), the system has hsould itself as being extremly stable and it can cope with an average number of user on the infrastructure that we already have in place. We are very happy about it.
Efficiency The system is overall very efficient, and does not use too many resources. Space wise, the project uses maximum 350MBs and RAM wise, an average of 600 MBs. The RAM cost might seem high, but this is only due to the contracts being loaded as strings into memory. RAM usage is not proportionate and will not scale if the system becomes larger, therefore not being an issue.
Compatibility The server application can run on Ubuntu and Mac (we do recoment Ubuntu 16 or alter though). System with at least 1 GB of RAM should be able to run the server. On the client side, there are no operating system or memory specifications, but the app does nto scale well on smaller screens (mobile is a problem) and there are some issues when using Internet Explorer and Mozzila Firefox. Compatibility definetly needs to be improved, but we did not consider it a priority for the purposes of this project.
Maintanability The server application requires almost no maintenance. Even the little that it does (cleaning backups, restarting etc.) is done extremly easy through the use of the Bash scripts included with the project.
Evaluation of Testing We have conducted extensive testing, including unit testing, integration testing, full system testing, automatic testing etc. We have fixed all bugs we found and we are overall very happy with the testing we have conducted.
Project Management Project Management was overall easy. We have used source versioning tools such as Git and GitHub (and GitKraken), and for documentation management we have used Google Drive and Google Docs.
